📅 [1주차] 주간 회의 아젠다: "설계 확정 및 가상 환경 구축"

1. 데이터 규격(Protocol) 최종 동기화
부품이 없어도 개발을 시작하려면, 서로 주고받을 데이터의 **'형식'**이 완벽히 약속되어야 합니다.

회의 내용: 제가 짜드린 API 명세서와 데이터 명칭(dustValue, noiseLevel, wifiDeviceCount 등)을 팀원 모두가 자기 코드에 변수명으로 쓰기로 확정합니다.

핵심 질문: "성재(백엔드)야, 이 JSON 형식으로 데이터 보내면 DB에 바로 꽂을 수 있게 테이블 만들 수 있지?" / "혁진(프론트)아, 이 이름으로 데이터 주면 바로 그래프 그릴 수 있지?"

2. 가상 데이터(Dummy Data) 생성 계획
센서가 없으니 코드로 만든 **'가짜 값'**을 사용하기로 합니다.

팀장(IoT) 역할: ESP32가 없더라도 PC에서 파이썬이나 간단한 툴로 **"10초마다 가짜 미세먼지/소음 데이터를 MQTT로 쏘는 프로그램"**을 만들어 백엔드에게 주기로 약속합니다.

효과: 이렇게 하면 백엔드와 프론트엔드는 센서가 오기 전에도 실시간 그래프가 움직이는 것을 확인할 수 있습니다.

3. 팀원별 환경 세팅 및 기술 스택 점검
모두가 개발 준비가 되었는지 확인합니다.

Frontend: React 프로젝트 초기 생성(CRA/Vite) 및 필요한 라이브러리(Axios, Chart.js, Tailwind CSS 등) 설치 확인.

Backend: Node.js Express 초기 설정 및 MQTT Broker(Mosquitto 등) 설치 완료 여부.

IoT: Arduino IDE 설치 및 ESP32 보드 매니저 설정 확인.

4. [리스크 관리] 교수님 피드백 반영 설계 (구조적 설계)
교수님이 말씀하신 '구조'를 어떻게 가져갈지 회의록에 남겨야 합니다.

안건: "우리 프로젝트는 데이터 유실을 막기 위해 MQTT 브로커를 중간에 두고, 백엔드는 데이터를 비동기적으로 처리하여 속도를 높이는 구조로 간다"는 점을 명확히 공유합니다.
