📗 Part 1. Notion: 우리 팀의 "공통 메모장" 팀원들이 가장 먼저 해야 할 일은 팀장님이 정성껏 만든 워크스페이스에 합류하는 것입니다.

가입 및 입장: 팀장님이 보낸 초대 이메일 확인 또는 공유 링크 클릭. Google 계정으로 간편 가입 권장.사용법 (이것만 알면 끝!):  


메인 대시보드 확인: 기획서 최종본과 기술 스택 확인하기. 
칸반 보드(업무 진행 상황): 본인 이름이 적힌 카드를 클릭해 내용을 확인하고, 작업 시작 시 **'진행 중'**으로, 완료 시 **'완료'**로 드래그하기. 
슬래시(/)의 마법: 빈 화면에 /를 치면 표, 이미지, 체크리스트 등 모든 기능을 꺼낼 수 있다고 알려주기.


<Github 실전가이드> - 코드 저장소
1. 자기 이름 브랜치에서 작업하기모든 팀원은 각자 컴퓨터에서 자기 이름 브랜치가 선택된 상태로 코드를 작성합니다. 
주의: main 브랜치에서 직접 수정하지 않도록 한 번 더 강조해 주세요. 
2. 코드 올리기 (Push)작업이 끝나면 깃허브로 코드를 올립니다. 이때는 오직 해당 팀원의 이름 브랜치에만 데이터가 쌓이므로 전체 프로젝트에는 영향을 주지 않습니다. 
3. 합쳐달라고 요청하기 (Pull Request)깃허브 웹사이트에 접속해 [본인 이름 브랜치] → [main] 방향으로 Pull Request를 생성하게 합니다. 
4. 팀장님의 최종 승인 (Merge)팀장인 동건 님이 코드를 확인하고 **[Merge pull request]**를 누르는 순간, 비로소 main 브랜치에 코드가 합쳐집니다. 

2. 요구사항 정의서, 와이어프레임 설계도, API 명세서 작성

3. AWS 서버 두 분중 한분이 만들어서 IAM 계정으로 초대해서 같이 서버 세팅 시작.
